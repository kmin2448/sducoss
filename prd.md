PRD: GAMJA HACK Live Scoreboard (데모 모델)
1. 프로젝트 개요
프로젝트명:GAMJA HACK Live Scoreboard
한 줄 설명:'Mini DEF CON Korea GAMJA HACK'에 참여하는 학생들을 위한 Jeopardy 방식의 실시간 CTF 점수판 쇼케이스 랜딩페이지.
유형:반응형 웹 랜딩페이지 및 실시간 대시보드 (프론트엔드 Mock-up)
난이도:보통 (백엔드 없이 프론트엔드 애니메이션 및 인터랙션 구현 집중)
2. 사용자 시나리오
누가:강원지역 대학 정보보호 동아리 학생 참가자 60명 및 행사 운영진.
언제:2026년 4월 24일(금) ~ 4월 25일(토) 행사 기간.
왜:글로벌 해킹 컨퍼런스인 DEF CON의 CTF 운영 방식을 직접 체험하며 현장감을 느끼고, 실시간으로 변동되는 점수판을 통해 순위 경쟁에 몰입하기 위해.
3. 핵심 기능 목록 (쇼케이스용)
필수 기능:
실시간 점수판 (Live Scoreboard): Jeopardy 방식 점수 체계가 적용된 팀별 가상 순위 노출.
문제 목록 및 상세 보기: 난이도/카테고리별 문제 격자형(Grid) 제공.
플래그(Flag) 제출 페이크 로직: 정답 입력 시 채점 및 점수 반영 시각 효과.
선택 기능 (전부 포함):
First Blood 하이라이트: 특정 문제를 최초로 푼 팀을 위한 알림 팝업.
실시간 순위 변동 그래프: 시간에 따른 가상 팀들의 점수 추이 차트 렌더링.
공지사항 티커 (Ticker): 대회 진행 관련 안내사항 롤링 텍스트.
4. 기술 스택
구현 방식:서버 연동 없는 프론트엔드 단독 구현 (모던 웹 기술 활용)
프론트엔드:HTML/CSS, JavaScript (React.js, Vue.js, 또는 Vanilla JS 활용 가능)
데이터:하드코딩된 JSON Mock Data 및 타이머(setInterval) 기반 가상 데이터 생성
라이브러리:Chart.js (순위 그래프), 순수 CSS/JS Animations (글리치, 네온 효과 등)
5. 화면 구성
상단 헤더 (GNB):로고, 깜빡이는 Live 인디케이터, 디바이스 연동 실시간 시계, 가상 로그인 상태 표기.
공지사항 티커:헤더 하단에 가상의 힌트 및 일정 공지가 흘러가는 무한 스크롤 영역.
좌측 메인 (대시보드):* 상단: 실시간 순위 변동 꺾은선 그래프
하단: 1~10위 실시간 리더보드 (순위 변동 애니메이션 포함)
우측 메인 (문제 보드):Web, Pwnable 등 카테고리별 Jeopardy 문제 타일 격자 배치.
모달 (팝업):문제 타일 클릭 시 노출. 문제 설명, 가짜 파일 다운로드 버튼, 플래그 입력 폼 포함.
하단 푸터:강원대학교 데이터보안활용혁신융합대학사업단, 강원정보보호지원센터, 국정원지부 로고 배치.
6. 상세 기능 명세 (인터랙션 중심)
Live 상태:초록색 원형 아이콘에 CSS pulse애니메이션을 적용하여 생동감 부여.
순위 변동 애니메이션:타이머 함수를 활용해 5~10초마다 가상 점수를 업데이트하고, 차트를 다시 그리며 리더보드 행(Row) 위치를 부드럽게 교체(Transition).
First Blood 팝업:우측 하단에 15~20초 간격으로 랜덤하게 등장 후 페이드 아웃(Fade-out) 처리.
플래그 제출 페이크 로직:* 플래그 입력 후 'Submit' 클릭 시 1.5초간 'Validating...' 로딩 스피너 작동.
이후 정답/오답 결과창 팝업 및 정답 시 해당 타일 'Solved' 상태로 실시간 전환 애니메이션.
7. 디자인 가이드
테마:사이버펑크 & 다크 테마 (Terminal, 해커톤 감성 극대화)
컬러 팔레트:
배경: #0D0D0D, #1A1A1A
포인트: #00FF41(네온 그린 / 정답, 활성화), #FF003C(네온 레드 / 오답), #0DE2EA(사이버 블루)
타이포그래피:* 숫자/코드: Fira Code또는 JetBrains Mono(Monospace 적용으로 숫자 폭 고정)
한글 텍스트: Pretendard또는 Noto Sans KR
시각 효과:네온 글로우(Neon Glow) 호버 효과, 간헐적 글리치(Glitch) 효과, 깜빡이는 터미널 커서(█), 각진 UI(Border-radius 0~2px).
8. 제약사항
데이터 한계:실제 DB 및 서버와 연동되지 않는 100% Mock Data 기반의 쇼케이스 모델입니다.
해상도 타겟:행사장(원주 오크벨리)의 대형 스크린 송출을 최우선으로 고려하여 데스크톱(1920x1080) 해상도에 UI/UX를 최적화합니다. 모바일 반응형은 후순위로 둡니다.
보안/인증:실제 참가자를 식별하는 회원가입, 세션, 플래그 검증 알고리즘 등은 구현하지 않습니다.
브라우저 환경:화려한 CSS3 효과 적용을 위해 Chrome, Edge 등 최신 모던 브라우저 환경에서 시연해야 합니다.
리소스 한계:주어진 예산안(총예산 6,050,000원) 내 개발비 미편성을 고려하여, 유료 에셋 없이 상업적 이용이 가능한 무료 오픈소스와 폰트만 활용하여 구축합니다.
